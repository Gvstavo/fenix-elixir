# Requisitos do Sistema

## Requisitos Funcionais

| ID | Requisito |
|---|---|
| RF01 | O sistema deve permitir que administradores gerenciem mangás, novels, capítulos, páginas, artistas, autores, gêneros e usuários. |
| RF02 | Usuários com a permissão **editor** devem gerenciar somente novels e mangás cuja permissão tenha sido concedida pelo administrador. |
| RF03 | O sistema deve permitir a criação de contas de usuário com a permissão padrão **leitor**. |
| RF04 | O sistema deve permitir que usuários favoritem mangás e novels. |
| RF05 | O sistema deve listar os 10 mangás mais lidos. |
| RF05 | O sistema deve listar as 10 novels mais lidas. |
| RF06 | O sistema deve contabilizar o número de acessos às páginas de mangás e novels. |
| RF07 | O sistema deve permitir a vinculação de capítulos a mangás e novels. |
| RF08 | O sistema deve permitir a vinculação de páginas aos capítulos. |
| RF09 | O sistema deve permitir a vinculação de autor(es), gênero(s) e artista(s) às novels e mangás. |
| RF10 | O sistema deve permitir a inclusão de imagens em formato **WebP** vinculadas aos capítulos de mangás. |
| RF11 | O sistema deve permitir que usuários façam o upload de uma foto de perfil |
| RF12 | Um capítulo de mangá é composto por uma série de imagens ordenadas: e.g. 001.webp, 002.webp
| RF12 | Um capítulo de novel é composto por uma série de rich-texts 

## Requisitos Não Funcionais

| ID | Requisito |
|---|---|
| RNF01 | O sistema deve ser web. |
| RNF02 | O sistema deve aplicar validação de papéis: **administrador**, **editor** e **leitor**. |
| RNF03 | O sistema deve ter um alto nível de usabilidade. |
| RNF04 | O sistema deve ser responsivo. |
| RNF05 | O sistema deve ser performático. |
